Mengganti Timezone di Ubuntu 18.04 Server

2019-01-24 :: Nixpoin

Pendahuluan⌗

Biasa, ketika kita membeli sebuah VPS atau Dedicated server maka jam pada server tadi sudah tersetting ke timezone yang berbeda dengan waktu di Indonesia. Sehingga kita kalau membaca log error cukup menyusahkan karena berbeda jam.

Mengecek Timezone di Server Ubuntu 18.04⌗

Untuk mengecek server kita menggunakan timezone apa, kita bisa gunakan perintah

<pre class="wp-block-preformatted">timedatectl

Hasilnya seperti

<pre class="wp-block-preformatted">Universal time: Kam 2019-01-24 03:41:21 UTC<br></br>         RTC time: n/a<br></br>        Time zone: UTC (EST, -0500)<br></br>      NTP enabled: n/a<br></br> NTP synchronized: no<br></br>  RTC in local TZ: no<br></br>       DST active: no<br></br>  Last DST change: DST ended at<br></br>                   Min 2018-11-04 01:59:59 EDT<br></br>                   Min 2018-11-04 01:00:00 EST<br></br>  Next DST change: DST begins (the clock jumps one hour forward) at<br></br>                   Min 2019-03-10 01:59:59 EST<br></br>                   Min 2019-03-10 03:00:00 EDT

Atau bisa juga menggunakan perintah

<pre class="wp-block-preformatted">cat /etc/timezone

Mengganti timezone dengan timedatectl⌗

Sebelum mengganti timezone, kita harus mencari tahu format timezone yang digunakan untuk daerah kita, Biasanya formatnya adalah Benua/Kotabesar yang bisa kita lihat di link berikut https://en.wikipedia.org/wiki/List_of_tz_database_time_zones#List

Karena kita berada di Indonesia, kita gunakan timezone Asia/Jakarta

Jalankan perintah di bawah ini untuk mengganti timezone kita menjadi ke Asia/Jakarta

<pre class="wp-block-preformatted">sudo timedatectl set-timezone Asia/Jakarta

Untuk mengecek apakah jam di server kita sudah berubah, jalankan lagi perintah

<pre class="wp-block-preformatted">timedatectl

atau perintah

<pre class="wp-block-preformatted">date
